##手机卫士APP

###模块一： 手机防盗
当手机丢失后，可以通过向丢失手机发送信息对已丢失的手机进行相应的提示，具备如下功能：

#*alarm*#"： 播放报警音乐
#*location*#： 向已经设置的安全手机发送手机的经纬度信息
#*lock*#: 锁屏并设置密码
#*wipedata*#: 清楚数据并关机（此功能在模拟器上并不能体现出来，但功能确实是有的）
#*uninstall*#： 卸载此应用

首先处理收到短信的逻辑代码：

// 1.1 监听手机是否收到短信
       Object[] objects = (Object[]) intent.getExtras().get("pdus");
       for (Object object : objects) {
           SmsMessage sms = SmsMessage.createFromPdu((byte[]) object);
            //1.2 获取短信的内容
           String message = sms.getMessageBody();
       ....
       }

####第一部分 播放报警音乐
//1.3 判断短信中是否包含#alarm#
if (message.contains(“#alarm#”)) {

    //1.4 如果包含则播放报警音乐 MediaPlayer
    MediaPlayer mediaPlayer = MediaPlayer.create(context, R.raw.ylzs);
    mediaPlayer.setLooping(true);
    mediaPlayer.start();
}

####第二部分 发送丢失手机的位置信息

//1.5 判断短信中是否包含#*location*#
   if (message.contains("#*location*#")) {

       //1.5.1 将获置取位置信息的功能放在服务中
       Intent service = new Intent(context, LocaitonService.class);
       context.startService(service);
   }

####获取设备管理权限的过程

参考网站如下：

https://developer.android.google.cn/guide/topics/admin/device-admin.html#lock
http://www.cnblogs.com/androidez/archive/2013/02/17/2915020.html

  1. 首先创建DeviceAdminReceiver的子类DeviceAdmin

    package com.example.lockscreen;

import android.app.admin.DeviceAdminReceiver;

public class DeviceAdmin extends DeviceAdminReceiver {

 }

    2.接着配置清单文件信息

  2. 接着在 res 目录下创建 xml 目录，并在其中创建名为 device_admin_sample 的文件：

    <device-admin xmlns:android="http://schemas.android.com/apk/res/android">
  <uses-policies>
<limit-password />
<watch-login />
<reset-password />
<force-lock />
<wipe-data />
<expire-password />
<encrypted-storage />
<disable-camera />
</uses-policies>
</device-admin>

    上面xml信息代表着设备管理策略
    ####第三部分 清除手机信息，恢复出厂设置

    DevicePolicyManager mDPM =
    (DevicePolicyManager)getSystemService(Context.DEVICE_POLICY_SERVICE);

    //2. 清除手机数据,恢复出厂设置

####第五部分 卸载此软件
根据收到的短信 进行卸载操作，此部分代码参考源码中的清单文件的配置，在广播接收器中进行修改。

源码的清单文件配置如下:

<activity android:name=".UninstallerActivity"
            android:configChanges="orientation|keyboardHidden"
            android:theme="@style/Theme.Transparent">
        <intent-filter>
            <action android:name="android.intent.action.VIEW" />
            <action android:name="android.intent.action.DELETE" />
            <category android:name="android.intent.category.DEFAULT" />
            <data android:scheme="package" />
        </intent-filter>
</activity>

在广播接收器中的代码如下:

Intent uninstall = new Intent("android.intent.action.DELETE");
uninstall.addFlag(FLAG_ACTIVITY_NEW_TASK);
//添加Category信息
uninstall.addCategory("android.intent.category.DEFAULT");

//添加Data信息
uninstall.setData(Uri.parse("package:"+context.getPackageName()));

context.startActivity(uninstall);

异常：
    Calling startActivity() from outside of an Activity  context requires the FLAG_ACTIVITY_NEW_TASK flag；
原因：
    Context中有一个startActivity方法，Activity继承自Context，重载了startActivity方法。如果使用Activity的startActivity方法，不会有任何限制，而如果使用Context的startActivity方法的话，就需要开启一个新的task，遇到上面那个异常的，都是因为使用了Context的startActivity方法；
解决方法：
    添加 uninstall.addFlag(FLAG_ACTIVITY_NEW_TASK)；
