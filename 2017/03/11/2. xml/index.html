#XML（Extensible Markup Language,可扩展标记语言）
用于数据存储，由于使用标签，层次比较清晰，而且比较严谨，

##1. XML语法 （标签/标记）

第一段：
    <网址>www..itcast.cn<网址>
第二段：
    <网址>
        www.itcast.cn
    <网址>  表示两个不同的数据，因为xml用于保存数据，两者保存的数据不同，因为后者包含换行和空格
区分大小写、不能以数字或“_”（下划线）开头、不能以xml（或XML、或Xml等）开头、不能包含空格、名称之间不能包含冒号（：）

##2. 数据解析（挖矿）
解析： 读取xml,将xml中的数据提取出来——
——Dom解析 （Document Object Model） 核心思想：把文档内容都封装成对象，保存在内存中，并且操作方便，又因为内存有限，会占用大量的内存。所以在使用增删改查比较频繁时使用，不适用于资源匮乏的设备上，（android）
（把矿里的东西都搬回来，结构也不会改变）

        ——Sax解析 原理：事件驱动，读取从头到尾，直到解析完整个文档
              优点：在读取xml文档中，已经根据定义好的事件，对xml内容进行了筛选，解析完成后，内存中保留了我们想要的内容，比较节约资源，在资源比较匮乏的平台使用
              缺点：没有保留文档的结构，无法进行增删改的操作；

             （告诉工人在遇到特定的矿石的时候进行不同的处理，进行筛选，但结构会改变）（手机、平板...）
        ···············································································
            sax中的事件：
                ·文档开始的事件： startDocument
                ·文档结束的事件： endDocument
                ·元素开始的事件： startElement
                ·文本时间：        character

        ——Pull解析（android官方指定）类似于Sax，都属于事件驱动的解析方式，但读取更灵活，以事件为单位，手动向下继续，如果获得我们要找的内容，就可以停止。
