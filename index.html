{
    if(sp == null)
    sp = context.getSharedPreferences("config", Context.MODE_PRIVATE);

    boolean aBoolean = sp.getBoolean(key, defValue);

    return aBoolean;
}

 *

 */
   public static void putBoolean(Context context,String key,boolean value)

##1. 计算机表示图形的几种方式：

###图形的大小 = 图片的总像素 每个像素的大小
如：800400像素的图片 320000/8 = 40000 byte

####-单色： （每个像素最多可以表示两种颜色），只需要使用长度为1的二进制位来表示，每个像素占1/8byte；

####-16色 ：（每个像素最多可以表示16种颜色） 0000 - 1111 只需要使用长度为4的位二进制表示 那么每个像素占1/2 byte ;

####-256色： （每个像素最多可以表示256种颜色） 0000 0000 - 1111 1111 只需要使用长度为8的位二进制表示 ,每个像素占1 byte ;

####-24位：rgb r 1byte, g 1byte, b 1byte 一个 像素占3个byte

####-在andoroid 下 采用ARGB A 表示透明度， 一个像素表示4byte

####-图片：24003200 ， 手机 320480 ，宽缩放 ：7，高缩放：6，所以按照宽缩放进行，

用于数据存储，由于使用标签，层次比较清晰，而且比较严谨，

##1. XML语法 （标签/标记）

第一段：
    <网址>www..itcast.cn<网址>
第二段：
    <网址>
        www.itcast.cn
    <网址>  表示两个不同的数据，因为xml用于保存数据，两者保存的数据不同，因为后者包含换行和空格
区分大小写、不能以数字或“_”（下划线）开头、不能以xml（或XML、或Xml等）开头、不能包含空格、名称之间不能包含冒号（：）

##2. 数据解析（挖矿）
解析： 读取xml,将xml中的数据提取出来——
——Dom解析 （Document Object Model） 核心思想：把文档内容都封装成对象，保存在内存中，并且操作方便，又因为内存有限，会占用大量的内存。所以在使用增删改查比较频繁时使用，不适用于资源匮乏的设备上，（android）
（把矿里的东西都搬回来，结构也不会改变）

        ——Sax解析 原理：事件驱动，读取从头到尾，直到解析完整个文档
              优点：在读取xml文档中，已经根据定义好的事件，对xml内容进行了筛选，解析完成后，内存中保留了我们想要的内容，比较节约资源，在资源比较匮乏的平台使用
              缺点：没有保留文档的结构，无法进行增删改的操作；

             （告诉工人在遇到特定的矿石的时候进行不同的处理，进行筛选，但结构会改变）（手机、平板...）

##Spark 开发指南

###简介

  1. Spark 主提供的主要的抽象概念是一个弹性的分布式数据（RDDS）集，它是一个元素集合，划分到集群的不同节点上，可以被并行操作。RDDS的创建可以从Hadoop文件系统（或者任何支持Hadoop的文件系统）上的一个文件开始，或者通过转换这个驱动程序中已存在的Scala集合而来。用户也可以使Spark持久化到一个RDD到内存中，使其能够在并行操作中被有效的重用。最后，RDDS能自动从节点故障中恢复。

  2. Spark提供的第二个抽象概念是共享变量，它可以在并行操作中使用。默认情况下，Spark通过不同节点的一些列任务来运行一个函数。它将每一个函数中用到的变量的拷贝传递到每一个任务中。有时候，一个变量需要在不同的任务之间、或者任务和驱动程序之间共享。Spark支持两种类型的变量共享：广播变量，可以在所有节点中缓存一个值；累加器，一个只能做加法的变量，例如计数器和求和。

###弹性分布式数据集RDDS
Spark 反复围绕的一个概念是弹性分布式数据集，它是一个有容错机制的元素集合，并且可以被并行操作。有两种创建RDDS的方法，并行化你的驱动程序的已有的集合，或者引用一个外部存储系统的数据集，例如一个文件系统，HDFS、HBase,或者任何可以提供一个Hadoop　InputFormat的数据源。

##Spark 运行架构

###1. 术语定义

  1. Application : Spark Application 和 Hadoop MapReduce中的类似，指的是用户编写的Spark用户程序，包含了一个 Driver 功能的代码和分布在集群中多个节点上运行的Executor代码；
  2. Driver : Spark中的Driver即运行上述Application的main()函数并且创建SparkContext，其中创建SparkContext的目的是为了准备Spark应用程序的运行环境。在Spark中由SparkContext负责和ClusterManager通信，进行资源的申请、任务的分配和监控等；当Executor部分运行完毕后，Driver负责将SparkContext关闭。通常用SparkContext代表Driver;
  3. Executor : Application 运行在worker节点上的一个进程，该进程负责运行Task ，并且负责将数据存储在内存或者磁盘上，每个Application 都有各自独立的一批Execuotr。

##1、向 SharedPreference 中插入数据

###1)、通过 Context 对象创建一个SharedPreferenceduixang 对象:
//name:shared..的名称，mode: 文件的操作模式
SharedPreference sharedPreference = Context.getSharedpreferences(name,mode);

###2)、得到一个Editor对象:
Editor editor = sharedPreference.editor();

###3)、向 Editor 中添加数据:
//向editor 里面以 key-value 的形式存放数据
editor.putString(key,value);

###4)、提交 Editor 对象 :
//最后就是通过 commit() 方法提交
editor.commit();

##2、从 SharedPreference 中取出数据：

###1)、通过 Context 对象创建一个SharedPreferenceduixang 对象:
//name:shared..的名称，mode: 文件的操作模式
SharedPreference sharedPreference = Context.getSharedpreferences(name,mode);

###2)、使用 SharedPreference 的 getString() 方法获取数据
//key 代表存入的键，deValue 代表默认的值，即当 key 不存在的时候使用此值
String userName = sharedPreference.getString(key,deValue);
Stirng passsWord = sharedPreference.getString(key,deValue);

##sharedPreference 是通过xml文件来作为数据存储；（—-用来存放一些标记性数据，或是一些设置信息—-典型的案例就是软件的欢迎界面，第一次安装软件时，会进入到欢迎界面，但以后再次进入时就不再出现，就是因为在第一次进入到软件时会更改一些变量，用于标记是否是第一次使用。所以如果在手机的设置中点击清除软件的数据和缓存后，再次进入页面，就会出现欢迎界面。。。。。。）
